# ユースケース

## 背景

Asana で自分に割り当てられたタスクを、日常のタスク管理に使っている OmniFocus.app へ取り込み、
OmniFocus 上で一元的に管理したい。Asana を都度開かずに、OmniFocus のレビューフローに乗せたい。

## アクター

- **利用者**: Asana ユーザーであり OmniFocus ユーザー（単一個人）。macOS 上で CLI を手動実行する。

## 主要ユースケース

### UC-1: 割り当てタスクを OmniFocus に取り込む

- 前提: Asana の PAT と対象ワークスペースが設定済み。OmniFocus が起動している。
- 操作: 利用者が `asana-omnifocus-sync` を実行する。
- 結果: Asana で自分に割り当てられた未完了タスクが、OmniFocus の取り込み先プロジェクトにタスクとして作成される。
  各 OmniFocus タスクの note には元タスクの URL と識別子（`asana_gid`）が記録される。

### UC-2: 取り込み済みタスクの更新を反映する

- 操作: Asana 側でタスク名や期日が変わった後、再度同期を実行する。
- 結果: 既に取り込み済みの OmniFocus タスク（同一 `asana_gid`）が更新される。重複は作られない。

### UC-3: 完了/担当解除を反映する

- 操作: Asana 側でタスクを完了、または自分の担当から外した後、再度同期を実行する。
- 結果: 対応する OmniFocus タスクが完了状態になる。

### UC-4: 実行前に変更内容を確認する（ドライラン）

- 操作: 利用者が `--dry-run` 付きで実行する。
- 結果: 取得したタスクと、作成/更新/完了の予定操作が表示されるのみ。OmniFocus は変更されない。

### UC-5: Asana プロジェクトを OmniFocus のタグとして反映する

- 前提: Asana タスクが 1 つ以上のプロジェクトに所属している。
- 操作: 利用者が同期を実行する。
- 結果: タスクの所属 Asana プロジェクトが、OmniFocus のタグとして当該タスクに付与される。
  タグはルートタグ（既定 `Asana`）の配下にプロジェクト名でネストされる。
  タスクが複数の Asana プロジェクトに所属する場合は、対応するタグが複数付与される。
  どのプロジェクトにも所属しないタスクには、ルートタグのみが付与される。

Asana 側で所属プロジェクトが変わった後に再実行すると、当該タスクの管理対象タグ（ルートタグ配下）が現在の所属に置き換わる。
利用者が手動で付けた他のタグ（コンテキストや場所など）は保持される。

### UC-6: OmniFocus で完了したタスクを Asana に反映する

- 前提: 取り込み済みの OmniFocus タスクがあり、Asana 側では未完了のまま残っている。
- 操作: 利用者が OmniFocus でそのタスクを完了したのち、再度同期を実行する。
- 結果: 対応する Asana タスクが完了状態（`completed=true`）になる。
  Asana 側が次回以降の取得対象から外れるため、OmniFocus に同じタスクが再作成されることはない。

完了のみを書き戻す。
OmniFocus で破棄（dropped）したタスクや、保留などの状態は Asana へ書き戻さない（後述の非対象）。

## 同期の向きと正本

タスクの作成・更新・タグ付けは Asana を正本とし、Asana → OmniFocus の一方向で反映する。
完了だけは双方向に伝わる。
Asana で完了すれば OmniFocus 側を完了し（UC-3）、OmniFocus で完了すれば Asana 側を完了する（UC-6）。

## 非対象（スコープ外）

- 完了以外の状態（OmniFocus の破棄・保留・defer など）の Asana への書き戻し。
  これらは Asana に対応する状態が無いため反映しない。OmniFocus で破棄しても Asana が未完了なら、次回の同期で OmniFocus に再作成されうる。
- launchd 等による自動・定期実行。
- 複数ユーザー・OAuth 認証。
- OmniFocus のタグ削除（空になった管理対象タグの掃除）。リネームや所属解除で使われなくなったタグは残す。
